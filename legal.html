🔒 Technical Security Measures
  • HTTPS everywhere — All data transmitted over encrypted SSL/TLS connections
  • JWT tokens — Secure session management with 7-day expiry for all portals
  • Bcrypt password hashing — Passwords never stored in plain text
  • PIN encryption — Staff PINs are encrypted before storage
  • Rate limiting — Login endpoints limited to 10 attempts per 15 minutes
  • Helmet.js — HTTP security headers to prevent common web attacks
  • CORS protection — API access restricted to authorized domains only
  • Row Level Security (RLS) — Database-level access control via Supabase
  • OTP verification — Phone verification required for new account creation
  • Device fingerprinting — Trial abuse prevention without invasive tracking
🖼️ Guest ID Photo Security

Guest ID photos uploaded during pre check-in are treated with the highest level of care:

  • Photos are uploaded directly to Supabase Storage over encrypted connections
  • Storage bucket access is controlled — only authenticated HOD portal can view photos
  • Photos are accessible via time-limited secure URLs, not permanent public links
  • Hotel owners can configure photo retention period in settings
  • Welco staff do not access guest ID photos under any circumstances

Legal Note for Hotels: Under Indian law (Foreigners Act, 1946 and Hotels and Lodging Houses Act), hotels are required to maintain guest identity records. Welco provides the digital infrastructure for this — hotels remain legally responsible for compliance with local registration requirements.

🏗️ Infrastructure Security
  • Supabase — SOC 2 Type II compliant database infrastructure
  • Render — Automated TLS certificates, DDoS protection
  • Environment Variables — All API keys stored as server-side secrets, never in code
  • Separation of concerns — Frontend and backend fully separated
